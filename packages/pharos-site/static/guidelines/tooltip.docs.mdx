## Usage

### Placement

There are 12 placement options available. Tooltips can be placed top, bottom, left or right of the target (e.g. button, icon, link, etc.) and the caret (triangle) can be placed at the start, center or end of the tooltip.

#### Where should I place the tooltip?

All tooltips, by default, are positioned top-center.

You can also decide to position the tooltip in a more appropriate place depending on the use case.

##### Navigational

For navigational items, the tooltip should be placed to the left of the associated icon button.

##### Icon toolbar

For content viewing experiences, the tooltip should sit below the viewer control icon buttons.

<img
  src={tooltipViewer}
  alt="Tooltip placement for viewer full screen"
  className="pharos-storybook__image"
  width="700"
/>

##### Icon buttons

For tooltips that are near other objects or items, display the tooltip to have more contrast, prominence and readability. In the example below, the tooltip has a top placement.

##### Longer titles

For titles that need a tooltip to display the full title, it should sit above the content, so the link text is still visible to the user.

##### Caret placement

We use [Popper.js](https://popper.js.org/) to position our tooltips. By default, tooltips are placed in the center of the trigger, but the caret/arrow will move and remain "attached" with Popper's overflow functionality.

### Longer content

Larger tooltips can be used to display further additional information. It is highly recommended that you consider if you need a tooltip or not for the design. "Throw it in a tooltip" is not the answer.

## Accessibility

### Guidance

1. Tooltips display when their trigger element receives keyboard focus or is hovered over with a mouse.
2. After the tooltip opens, initial focus should be set on the first focusable element in the tooltip.
3. After the tooltip closes, focus should remain on the trigger.
4. Focus must not move outside the tooltip's trigger until a user dismisses it via keyboard tabbing or moving their mouse off the trigger.
5. A tooltip can be dismissed when a user moves their mouse off the trigger, removes keyboard focus of the element that invoked it, or by pressing the <kbd>Escape</kbd> key.

### Usability

#### Dismissible

A user can dismiss the tooltip without moving their pointer hover or keyboard focus. This must be able to work with:

- Mouse
- Keyboard
- Screen readers
- Magnification and zoom

#### Hoverable

If pointer hover can trigger the additional content, then the pointer can be moved over the tooltip without it disappearing.

_Tooltip remains open while the cursor moves within the trigger and disappears when the cursor moves out of the trigger._

#### Persistent

The additional content remains visible until the hover or focus trigger is removed, the user dismisses it, or its information is no longer valid.

### Keyboard interactions

Pressing <kbd>Escape</kbd> dismisses the tooltip.

### Roles

- The element that serves as the tooltip container has `role="tooltip"`.
- The element that triggers the tooltip references the tooltip element with aria-describedby.
- Don't rely on `title` attributes. They are not keyboard accessible and are not supported in many screen reader setups.

## Content guidelines

**Tooltips should:**

- Be concise and scannable
- Be written in sentence case
- Not be used to communicate error messages or critical info
